Why rqlite

When SQLite isn't enough and Postgres is too much.

SQLite is the most-deployed database engine in the world: a single file, no server, no fuss. But a single file is fragile — lose the disk and you lose the data. rqlite keeps SQLite's simplicity and layers Raft-based replication on top, so the same database lives on three or five nodes and any one of them can fail without taking the cluster down.

Postgres is the right answer when you need write-scaling and a deep ecosystem. It's heavy when you just need durable, replicated relational storage for configuration, control-plane data, or moderate workloads. rqlite ships as a single static binary, forms a cluster with one flag, and asks for nothing else — no external coordinator, no separate consensus layer.
